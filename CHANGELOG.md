# Changelog

## v0.2.5 (2026-06-22)

### 新增：智能文档生成编排系统

完整的多阶段文档生成流水线，替代原有单轮 AI 调用模式：

- **6 阶段状态机**：素材分析 → 大纲规划 → 风格锚定 → 并行生成 → 终审检查 → 定向修复
- **3 个用户检查点**：大纲确认、风格确认、终审确认，每个检查点支持返回修改
- **并行生成**：多个专题同时生成，信号量控制并发，单专题失败不影响其他
- **一键返工**：大纲质检发现问题时，一键将问题反馈给 AI 重新生成
- **进度推送**：实时显示当前阶段、专题级进度、质检结果

### 新增：智能模式 UI

- 自适应三栏布局：生成时左栏可折叠，右栏显示阶段上下文
- 检查点模态框：大纲/风格/审阅结果的专用确认面板
- 补充说明历史：localStorage 持久化历史指令，支持快速复用
- ErrorBoundary：渲染错误不再白屏，显示错误信息和重试按钮

### 改进

- JSON 解析鲁棒性：尾逗号修复、连续双引号处理、正则兜底
- IPC 载荷优化：`sanitizeSnapshot` 剥离大字段防止渲染进程崩溃
- 文档质量：Phase 3 提示词增加格式一致性、答案完整性、禁止事项约束
- Phase 5 输出：增大 maxTokens 至 32768，去除 AI 前言文本
- MaterialPicker 尺寸：改为 `flex-1 min-h-0`，自适应容器高度
- 项目命名统一：`final-exam-completer` → `final-exam-closure`

### 修复

- 返回修改时 `Assignment to constant variable` 崩溃
- 检查点模态框 z-index 被滚动容器遮挡
- 两个编排实例同时启动导致白屏
- `fullDocument`（158K-353K 字符）通过 IPC 传输导致渲染进程崩溃
- Phase 5 输出被截断，AI 生成前言文本混入文档
- JSON 解析失败导致编排中断
- CheckpointReview 未知 severity 类型导致渲染崩溃
