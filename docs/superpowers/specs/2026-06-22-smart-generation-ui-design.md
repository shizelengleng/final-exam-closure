# 智能文档生成 UI 设计规格

## 概述

重新设计 ContentGenerator 的智能模式 UI，解决当前布局空间利用率低、检查点交互缺失、"返回修改"直接中断流程等问题。

## 设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 整体布局 | 自适应三栏 + 模态检查点 | 左侧折叠给主区更多空间，Modal 焦点集中 |
| 返回修改 | Modal 内嵌输入框 | 不离开 Modal，流程不中断 |
| 右侧面板 | 按阶段自动切换 | 每个阶段提供最相关的上下文 |
| 结果展示 | 原地切换为文档预览 | 与普通模式一致，平滑过渡 |
| 性能策略 | CSS transition + fixed positioning | 避免 JS 驱动动画，减少重排 |
| 交互区域 | Modal 80%+ 高度，滚动区最小 300px | 确保触摸友好，内容不挤压 |

## 布局结构

### 三栏自适应

```
┌─────────────────────────────────────────────────────────┐
│                      Header                              │
├──────┬──────────────────────────────────┬───────────────┤
│      │                                  │               │
│ Left │          Center                  │    Right      │
│      │                                  │               │
│ 48px │         flex-1                   │   280px       │
│(折叠)│                                  │               │
│      │                                  │               │
└──────┴──────────────────────────────────┴───────────────┘
```

### 左侧面板（折叠态）

- 宽度：48px
- 内容：3 个图标按钮（模式/模板/智能模式）
- 生成完成后恢复为完整面板（w-72）

### 中间面板（主内容区）

- 占满剩余空间（flex-1）
- 根据状态切换内容：
  - 普通模式：Markdown 预览 + 版本切换
  - 智能生成中：进度视图
  - 智能生成完成：Markdown 预览

### 右侧面板（上下文区）

- 宽度：280px
- 按阶段自动切换内容（见下方）

---

## 状态机

```
idle → generating → checkpoint → generating → ... → completed
                ↓                    ↑
            (return to modify) ------┘
```

### 状态定义

| 状态 | 左侧 | 中间 | 右侧 |
|------|------|------|------|
| idle | 完整配置面板 | 空态/预览 | 对话面板 |
| generating | 折叠图标栏 | 进度视图 | 阶段上下文 |
| paused_checkpoint | 折叠图标栏 | 进度视图 + Modal | 阶段上下文 |
| completed | 恢复完整面板 | 文档预览 | 对话面板 |

## 检查点 Modal

### 布局

```
┌─────────────────────────────────────────────┐
│ 📋 确认文档大纲            8 个专题 · 12 页  │
├─────────────────────────────────────────────┤
│                                             │
│ 《数据结构》期末复习系统指南                  │
│ 系统化的复习资料，涵盖所有核心知识点          │
│                                             │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│ │ 1. 线性  │ │ 2. 树与图│ │ 3. 排序  │     │
│ │ 数组链表 │ │ 二叉树   │ │ 冒泡快排 │     │
│ │ 栈队列   │ │ BFS/DFS  │ │ 归并     │     │
│ └──────────┘ └──────────┘ └──────────┘     │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│ │ 4. 查找  │ │ 5. 动态  │ │ 6. 图论  │     │
│ │ ...      │ │ ...      │ │ ...      │     │
│ └──────────┘ └──────────┘ └──────────┘     │
│                                             │
├─────────────────────────────────────────────┤
│ [跳过]              [返回修改]  [确认大纲]   │
│                                             │
│ ┌─ 修改意见（展开后显示）─────────────────┐ │
│ │ 请输入修改意见...                       │ │
│ │                                         │ │
│ │ [取消]              [重新生成大纲]       │ │
│ └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

### 规格

- 宽度：屏幕 85%
- 高度：屏幕 80%+（最小 500px）
- 滚动区域最小高度：300px
- 按钮最小触摸区：44px
- 背景遮罩：rgba(0,0,0,0.45)
- 动画：fade-in + scale-up（CSS transition）

### 三种检查点

| 检查点 | 标题 | 内容 | 按钮 |
|--------|------|------|------|
| 1. 大纲确认 | 确认文档大纲 | 专题网格卡片 | 跳过 / 返回修改 / 确认 |
| 2. 风格确认 | 确认内容风格 | 首个专题 Markdown 预览 | 跳过 / 返回修改 / 确认 |
| 3. 审阅确认 | 确认修复方案 | 问题列表 + 建议 | 跳过修复 / 确认修复 |

### 返回修改交互

1. 用户点「返回修改」→ Modal 底部展开输入框
2. 用户输入修改意见（可选）
3. 点「重新生成」→ 当前阶段重新运行
4. Modal 保持打开，显示新进度
5. 新结果替换旧内容，用户再次确认

## 右侧面板内容（按阶段切换）

| 阶段 | 显示内容 |
|------|----------|
| Phase 0 | 知识点提取进度（已提取 N 个） |
| Phase 1 | 知识点列表（可展开） |
| Phase 2 | 大纲预览（专题列表） |
| Phase 3 | 专题生成状态（完成/进行中/待做） |
| Phase 4 | 质检结果摘要 |
| Phase 5 | 修复进度 |
| completed | 对话修改面板 |

---

## 性能要求

- 左侧折叠动画：CSS transition，200ms ease
- Modal 弹出：CSS animation，150ms
- 进度条动画：CSS animation，避免 JS 驱动
- Modal 使用 `position: fixed`，避免触发父组件重排
- 进度事件节流：最多每 500ms 更新一次 UI
