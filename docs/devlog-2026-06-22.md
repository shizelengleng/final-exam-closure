## 一、智能文档编排器 (`documentOrchestrator.ts`)

### 架构

6 阶段状态机，每阶段独立 prompt + 质检 + 重试：

```
Phase 0: 素材分析 → 知识点列表（JSON 解析 + 字段校验）
Phase 1: 大纲规划 → 文档计划（5 项代码自查 + Checkpoint 1）
Phase 2: 风格锚定 → 首个专题（长度/标题/练习校验 + Checkpoint 2）
Phase 3: 并行生成 → 逐专题（信号量并发 + 每专题质检 + 重试 2 次）
Phase 4: 终审检查 → 问题列表（API review + Checkpoint 3）
Phase 5: 定向修复 → 最终文档（增量修复 + 长度校验）
```

### 关键设计

- **Checkpoint 暂停/恢复**：`Promise` + `resolve` 模式，renderer 通过 `orchestrator:resume` 恢复
- **`sanitizeSnapshot`**：IPC 传输前剥离 `fullDocument`、`topicContents[].content`、`styleAnchor.content` 等大字段
- **并发控制**：信号量限制 Claude CLI 并发为 2，REST API 为 3
- **JSON 解析鲁棒性**：尾逗号修复、连续双引号 `""` → `\"`、正则兜底提取数组

## 四、稳定性修复

### 白屏问题（共 4 个根因）

1. **IPC 载荷过大**：`fullDocument` 158K-353K 字符通过 IPC 传输 → `sanitizeSnapshot` 剥离
2. **const 重赋值**：`plan`/`anchor` 声明为 `const`，返回修改时崩溃 → 改为 `let`
3. **双实例启动**：快速点击或异步竞争导致两个编排同时运行 → ref guard + main 进程检查
4. **渲染错误未捕获**：组件内部异常导致整个 React 树卸载 → `ErrorBoundary`

### JSON 解析

- `cleanJsonString`：尾逗号修复、未转义控制字符处理、连续双引号修复
- `parseJsonFromAI`：正则兜底提取 JSON 数组（当 AI 输出包含前言文本时）
- `aiClient.ts` 中的 `cleanJsonString` 同步修复

### Phase 5 截断

- `maxTokens` 16384 → 32768
- `noContinue: true` 禁止自动续写
- 后处理剥离 AI 前言（找到第一个 `\n#` 标题作为文档起点）
