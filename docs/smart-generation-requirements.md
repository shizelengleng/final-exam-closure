# 智能文档生成 - 需求与 UI 布局规划

## 一、功能概述

智能文档生成是 ContentGenerator 的增强模式，通过多阶段 AI 编排生成高质量复习文档。与普通模式（单轮生成）相比，智能模式提供：
- 渐进式生成（6 个阶段）
- 每阶段质量检查
- 用户确认检查点（3 个）
- 中途可干预

---

## 二、6 阶段流程

| 阶段 | 名称 | 耗时 | 输出 | 是否需要用户交互 |
|------|------|------|------|------------------|
| 0 | 素材分析 | ~30s | 知识点列表（JSON） | 否 |
| 1 | 大纲规划 | ~30s | 文档大纲（JSON） | **是 — 确认大纲** |
| 2 | 风格锚定 | ~30s | 首个专题全文 | **是 — 确认风格** |
| 3 | 并行生成 | ~2-5min | 所有专题内容 | 否（每专题有进度） |
| 4 | 终审检查 | ~30s | 问题列表（JSON） | **是 — 确认修复方案** |
| 5 | 修复输出 | ~30s | 最终文档 | 否 |

## 四、UI 功能需求清单

### 4.1 主界面布局（ContentGenerator 左中右三栏）

**左侧面板（配置区）：**
- 模式选择卡片（文档/文章/视频）
- 模板选择
- 补充说明输入
- 智能模式开关
- 生成按钮
- 资料选择器

**中间面板（主内容区）：**
- 普通模式：Markdown 预览 + 版本切换
- 智能模式：需要重新设计（见 4.2）

**右侧面板（对话区）：**
- 普通模式：对话修改
- 智能模式：可能需要不同的交互

### 4.2 智能模式中间面板 — 需要设计的 UI

#### A. 整体进度视图
- 6 阶段横向步进器（当前阶段高亮）
- 总耗时计时器
- 取消按钮

#### B. 阶段详情区域
- 当前阶段名称 + 描述
- 阶段内进度条（Phase 3 并行生成时显示 "专题 3/8"）
- 质检结果（通过/警告/失败 Tag）

#### C. 检查点确认面板（3 种）

**Checkpoint 1 — 大纲确认：**
- 文档标题 + 概述
- 专题列表（编号、标题、子主题、练习策略）
- 按钮：确认大纲 / 返回修改
- "返回修改" 需要：显示输入框让用户说明修改意见 → 重新生成 Phase 1

**Checkpoint 2 — 风格确认：**
- 首个专题的 Markdown 预览（可滚动）
- 字数统计 + 结构元素数量
- 按钮：确认风格 / 返回修改
- "返回修改" 需要：输入框 → 重新生成 Phase 2

**Checkpoint 3 — 审阅确认：**
- 问题列表（严重/警告/提示 分级显示）
- 每个问题有描述 + 建议修复
- 按钮：确认修复 / 跳过修复
- 可选择性勾选要修复的问题

#### D. 最终结果视图
- 生成完成提示
- 阶段完成统计
- 保存到 Wiki / 导出 Markdown / 导出 PDF 按钮
- 文档预览

### 4.3 交互流程

```
用户点击「智能生成」
    ↓
中间面板切换为 OrchestrationProgress
    ↓
Phase 0 运行（显示进度）
    ↓
Phase 1 运行（显示进度）
    ↓
Checkpoint 1 弹出大纲确认面板
    ├→ 用户点「确认大纲」→ 继续 Phase 2
    └→ 用户点「返回修改」→ 显示输入框 → 用户输入意见 → 重新运行 Phase 1
    ↓
Phase 2 运行（显示进度）
    ↓
Checkpoint 2 弹出风格确认面板
    ├→ 用户点「确认风格」→ 继续 Phase 3
    └→ 用户点「返回修改」→ 显示输入框 → 重新运行 Phase 2
    ↓
Phase 3 运行（显示逐专题进度）
    ↓
Phase 4 运行（显示进度）
    ↓
Checkpoint 3 弹出审阅确认面板
    ├→ 用户点「确认修复」→ Phase 5 修复
    └→ 用户点「跳过修复」→ 直接输出最终文档
    ↓
显示最终文档 + 操作按钮
```

## 五、UI 设计要点

### 5.1 空间利用
- 中间面板是核心区域，需要最大化利用
- 检查点确认面板应该占据主要空间，而不是浮窗
- 进度信息应该紧凑，不占用太多垂直空间

### 5.2 视觉层次
- 阶段步进器：顶部横向，清晰显示当前位置
- 检查点面板：中间大区域，突出显示
- 操作按钮：底部固定，容易点击

### 5.3 状态反馈
- 每个阶段开始/完成都应该有视觉反馈
- 质检结果用颜色区分（绿色通过、黄色警告、红色失败）
- 耗时显示让用户有预期

### 5.4 响应式
- 左侧面板折叠时中间面板应该扩展
- 检查点面板在小屏幕上应该可滚动
